Design Philosophy

Content First

Activities and timer take priority in visual hierarchy. Everything else is secondary.

Functional Minimalism

Show only essential information. Hide complexity until it's needed.

Contextual Relevance

UI elements appear only when they provide actionable value to the user.

Progressive Enhancement

Mobile-first design with enhanced experiences on larger screens.

Best Practices

Component Guidelines

  • ✓ Use design tokens (bg-card, text-foreground) over specific colors
  • ✓ Follow mobile-first responsive patterns (max-w-md, then md:)
  • ✓ Include data-testid attributes for reliable testing
  • ✓ Use content-card class for consistent card styling
  • ✓ Apply hover states with 200ms transitions

Performance Considerations

  • → Minimize DOM complexity in activity cards
  • → Use CSS transforms over layout changes for animations
  • → Avoid animating properties that trigger reflow
  • → Keep transition durations under 300ms for responsiveness
